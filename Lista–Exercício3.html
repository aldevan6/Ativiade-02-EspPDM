1. Qual é a diferença entre linguagem de programação Server Side e Client Side?

A diferença entre linguagem de programação Server Side e Client Side é a seguinte:

Server Side

  • As linguagens server-side são executadas no lado do servidor.
  • Elas são processadas pelo servidor antes de enviar uma resposta para o navegador do usuário.
  • Exemplos de linguagens server-side são PHP, ASP, Java, Ruby, entre outras.
  • Essas linguagens são usadas para manipular a lógica principal da aplicação, como verificar se o usuário está logado, buscar informações no banco de dados, etc.
  • Elas são mais seguras para acessar um banco de dados, pois o código não é visível para o usuário.
  • Os desenvolvedores que trabalham com linguagens server-side são chamados de desenvolvedores back-end.
  • Fonte: gigasystems.com.br

Client Side

  • As linguagens client-side são executadas no navegador do usuário.
  • A linguagem client-side mais comum é o JavaScript.
  • Essas linguagens permitem a manipulação da página do usuário em tempo real, como alterar o valor de um campo de formulário ou criar elementos dinâmicos.
  • O código client-side é visível para o usuário e pode ser manipulado por ele.
  • É mais adequado para realizar validações e melhorias de usabilidade no navegador do usuário.
  • Os desenvolvedores que trabalham com linguagens client-side são chamados de desenvolvedores front-end.
  • Fonte: pt.wikipedia.org, tableless.github.io, pt.stackoverflow.com

Em resumo, a principal diferença é que as linguagens server-side são executadas no servidor e as linguagens client-side são executadas no navegador do usuário.
